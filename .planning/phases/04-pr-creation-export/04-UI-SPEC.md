## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `p-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing, modal body padding (`gap-4`, `p-4`) |
| lg | 24px | Section padding, modal section gaps (`gap-6`, `p-6`) |
| xl | 32px | Layout gaps (`gap-8`) |
| 2xl | 48px | Major section breaks |

## Component Inventory

### 1. GitHub PR Tab (in existing ExportModal)

**Location:** New tab added to `packages/ui/components/ExportModal.tsx` existing tab bar.

**Tab label:** "GitHub PR"

**Tab content states:**

| State | What user sees |
|-------|---------------|
| Not authenticated | "Sign in with GitHub to create pull requests." + "Sign in with GitHub" button (existing pattern from ExportModal Share tab) |
| Authenticated, no PR | Annotation count summary + "Export to GitHub PR" primary button |
| Authenticated, PR exists | PR link with "View PR" button + annotation count summary + drift warning (if applicable) |
| Exporting | "Export to GitHub PR" button disabled with "Creating PR..." label + spinner |
| Export success | Toast appears (see Toast below), tab shows PR link |
| Export error | Toast appears with "Retry" action (see Toast below) |

**Annotation count display:**
- Text: "{N} annotations will be exported as review comments"
- If N = 0: "No annotations to export" (muted text, export button disabled)
- If annotations have images: Warning line below count: "Annotations with images will be exported as text only"
- Style: `text-xs text-muted-foreground`

**Drift warning (shown when plan hash differs from stored PR hash):**
- Container: `p-2 bg-warning/10 border border-warning/20 rounded-md`
- Icon: Warning triangle SVG (same as ConfirmDialog warning variant)
- Text: "Plan changed since PR was created -- line numbers may be incorrect"
- Style: `text-xs text-warning`
- Position: Above annotation count, below PR link

**"Export to GitHub PR" button:**
- Style: `w-full px-3 py-2 rounded-md text-xs font-medium bg-primary text-primary-foreground hover:opacity-90 transition-opacity`
- Disabled style: `disabled:opacity-50 disabled:cursor-not-allowed`
- Loading label: "Creating PR..."
- Matches existing "Create Authenticated Share" button pattern exactly

### 2. Toast Notifications

**Implementation:** Extend existing `noteSaveToast` pattern in App.tsx. Add optional `action` field to toast payload.

**Toast payload type extension:**
```typescript
type ToastPayload = {
  type: 'success' | 'error';
  message: string;
  action?: { label: string; onClick: () => void };
} | null;
```

**Success toast:**
- Message: "PR created successfully"
- Action: `{ label: "View PR", onClick: () => window.open(prUrl, '_blank') }`
- Duration: 6 seconds auto-dismiss
- Style: existing `bg-success/15 text-success border border-success/30`

**Error toast:**
- Message: Error from API (e.g., "Failed to create PR: permission denied")
- Action: `{ label: "Retry", onClick: retryExport }`
- Duration: 8 seconds auto-dismiss
- Style: existing `bg-destructive/15 text-destructive border border-destructive/30`

**Retrying toast:**
- Message: "Retrying... (attempt 2 of 3)"
- No action button
- Duration: persists until retry completes
- Style: `bg-muted text-muted-foreground border border-border`

**Toast action button style:**
- `ml-2 px-2 py-0.5 rounded text-xs font-medium bg-background border border-border hover:bg-muted transition-colors`
- Inline with toast message, right-aligned

**Toast position:** `fixed top-16 right-4 z-50` (matches existing toast position)

### 3. No New Modals or Dialogs

The Export Modal GitHub PR tab handles all states inline. No confirmation dialog needed before export (D-02: one-click PR creation). No separate PR creation modal.

## Interaction Contracts

### Export Flow (happy path)

1. User opens Export Modal (existing toolbar button)
2. User clicks "GitHub PR" tab
3. Tab shows annotation count + "Export to GitHub PR" button
4. User clicks "Export to GitHub PR"
5. Button changes to "Creating PR..." (disabled)
6. On success: Toast appears with "PR created successfully" + "View PR" action
7. Tab updates to show PR link + annotation count

### Export Flow (error + retry)

1. Steps 1-5 same as happy path
2. On error: Toast appears with error message + "Retry" action
3. Button reverts to "Export to GitHub PR" (enabled)
4. User clicks "Retry" in toast OR clicks button again
5. Toast changes to "Retrying... (attempt 2 of 3)"
6. On success: follow happy path step 6

### Export Flow (auth expired)

1. Steps 1-5 same as happy path
2. On 401: Toast appears with "Authentication expired. Please sign in again."
3. After 2 seconds, redirect to OAuth login with return_to URL (D-19)

### Drift Warning Flow

1. User opens Export Modal "GitHub PR" tab
2. If PR metadata exists AND plan hash differs from stored hash:
   - Warning banner appears above annotation count
   - Export button remains enabled (non-blocking per D-15)
3. User can proceed with export despite warning
