  <action>
The hook build uses `viteSingleFile()` to inline ALL dependencies into a single HTML file. React is NOT external -- it gets bundled. The problem is that when Vite resolves files inside `packages/github/client/`, Rollup cannot find `react/jsx-runtime` because `packages/github/` only lists React as a `peerDependency` and has no `node_modules/react/`.

Two changes are needed:

**Change 1: Add `resolve.dedupe` to apps/hook/vite.config.ts**

Add `resolve.dedupe: ['react', 'react-dom']` to the Vite config. This tells Vite to always resolve React from a single location (the hook app's node_modules), regardless of which package the import originates from. This is the standard Vite solution for monorepos with peer dependencies.

```typescript
resolve: {
  dedupe: ['react', 'react-dom'],
  alias: {
    // ... existing aliases unchanged
  }
}
```

**Change 2: Add `@plannotator/github` as workspace dependency in apps/hook/package.json**

Add `"@plannotator/github": "workspace:*"` to the dependencies section. This ensures Bun's package resolution links the workspace and Vite can traverse the dependency graph correctly. Then run `bun install` to update the lockfile.

**Change 3 (if Change 1+2 alone do not fix): Broaden the alias**

If the build still fails after adding dedupe, change the `@plannotator/github/client` alias from pointing to a specific `.tsx` file to pointing to the directory:

```typescript
'@plannotator/github/client': path.resolve(__dirname, '../../packages/github/client'),
```

This allows Vite to resolve sub-imports (like `react/jsx-runtime`) using the standard module resolution algorithm from the directory rather than treating the alias as a single-file redirect.

After changes, run:
1. `bun install` (to update lockfile with new workspace dep)
2. `bun run build:hook` (must exit 0)
3. `bun run --cwd apps/review build` (must still exit 0, no regression)
