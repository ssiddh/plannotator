# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Assessment

**Phase 1 is a backend/structural phase with zero user-visible UI changes.** The work extracts scattered GitHub integration code from `apps/paste-service/` into `packages/github/` and adds a React Context Provider wrapper (`<GitHubProvider>`) to `packages/editor/App.tsx`. This wrapper is invisible to the user -- it provides context state and stubbed action methods consumed by future phases (Phase 2+ for auth UI, Phase 4+ for sync buttons).

No new components, screens, modals, buttons, or visual elements are introduced. The existing design system (18 themes, CSS custom property tokens, Tailwind v4 bridge in `packages/ui/theme.css`) is unchanged.

This UI-SPEC documents the existing design system contract so downstream phases (which DO introduce UI) have a baseline reference.

## Color

Phase 1 introduces no new color usage. The existing token system is documented here as the baseline contract for future phases.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` | Page background, surfaces |
| Secondary (30%) | `var(--card)` / `var(--muted)` | Cards, sidebar, popover, input backgrounds |
| Accent (10%) | `var(--accent)` | Feedback button highlight, annotation markers |
| Destructive | `var(--destructive)` | Deny/delete actions only |

Additional semantic tokens already established:
- `--primary`: CTA buttons (Approve), focus ring
- `--success`: Active/online indicators, approve states
- `--warning`: Caution states
- `--border`: All element borders
- `--muted-foreground`: Deemphasized text

Accent reserved for: feedback button borders and backgrounds (`bg-accent/15`, `border-accent/30`), annotation highlight markers, send feedback CTA

**Source:** `packages/ui/themes/plannotator.css` (all token definitions), `packages/ui/theme.css` (Tailwind bridge), `packages/ui/components/ToolbarButtons.tsx` (usage patterns)

## Phase 1 Specific: GitHubProvider Contract

While invisible to users, the `GitHubProvider` wrapper has a technical contract that affects future UI phases:

### Context Shape (Stubs for Phase 1)

```typescript
interface GitHubContextValue {
  // Auth state (Phase 2 populates these)
  isAuthenticated: boolean;    // default: false
  user: GitHubUser | null;     // default: null
  token: string | null;        // default: null (reads localStorage on mount)

  // PR state (Phase 4 populates these)
  prMetadata: PRMetadata | null; // default: null

  // Actions (stubs in Phase 1, throw "not implemented" or no-op)
  syncFromGitHub: () => Promise<void>;
  syncToGitHub: () => Promise<void>;
  createPR: () => Promise<void>;
}
```

### Rendering Contract

- Provider MUST NOT render any visible DOM elements
- Provider MUST NOT alter layout, spacing, or visual hierarchy
- Provider MUST NOT introduce loading states or spinners
- Provider MUST wrap children transparently: `<GitHubContext.Provider value={value}>{children}</GitHubContext.Provider>`
