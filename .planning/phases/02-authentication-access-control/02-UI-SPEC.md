# Phase 2 — UI Design Contract

> Visual and interaction contract for the authentication & access control phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (inline HTML — server-rendered 401 page) |
| Icon library | inline SVG (GitHub Octocat logo) |
| Font | system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif |

**Rationale:** The UI surface for this phase is a single server-rendered HTML error page (not a React component). Per D-01 and D-05, the auth-required page is returned as inline HTML in a 401 response before any client-side JS loads. It cannot use the Plannotator theme system or React because the plan app is never served. The existing `Login.tsx` in the portal uses the same inline-style pattern with `system-ui` font stack.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Sign in with GitHub" |
| Auth-required heading | "Authentication Required" |
| Auth-required body | "This share requires GitHub authentication." |
| Session expired heading | "Session Expired" |
| Session expired body | "Your session has expired. Please sign in again to continue." |
| Session expired CTA | "Sign in with GitHub" |
| Access denied heading | "Access Denied" |
| Access denied body | "You don't have permission to view this share. Contact the share owner to request access." |
| Legal disclaimer | "By signing in, you agree to allow Plannotator to access your GitHub profile and organization membership." |

**States and their HTTP status codes:**

| State | Status | When |
|-------|--------|------|
| Auth required (no token) | 401 | Whitelist share, user not authenticated |
| Session expired (invalid token) | 401 | Token validation returns `valid: false` (D-11) |
| Access denied (valid token, not on ACL) | 403 | User authenticated but not in whitelist users/teams |

**Empty state:** Not applicable — this phase has no data-bearing views.

**Destructive actions:** None in this phase.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Button hover | Background transitions from `#24292e` to `#2c3136` over 200ms |
| Button click | Navigates to `/api/auth/github/login?return_to={current_url}` |
| Focus visible | 2px solid outline, 2px offset (standard accessibility ring) |
| OAuth redirect | After GitHub auth, user redirected back to original share URL (D-08) |
| Page load (public share) | Auth page never shown — public shares skip auth entirely (D-02) |

**Accessibility requirements:**
- Button must have `role="link"` or be an `<a>` element (it triggers navigation, not form submission)
- Page must have `<title>Authentication Required</title>`
- GitHub icon must have `aria-hidden="true"` (decorative)
- Color contrast: heading `#111` on `#fff` = 18.9:1 (AAA), body `#666` on `#fff` = 5.7:1 (AA), button `#fff` on `#24292e` = 13.9:1 (AAA)
