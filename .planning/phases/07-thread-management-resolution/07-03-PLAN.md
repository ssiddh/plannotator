  <action>
    Create `packages/ui/hooks/useReview.ts`:

    ```typescript
    export type ReviewEvent = 'APPROVE' | 'REQUEST_CHANGES' | 'COMMENT';
    export type ReviewState = 'idle' | 'syncing' | 'submitting' | 'success' | 'error';

    export function useReview(options: {
      prMetadata: PRMetadata | null;
      githubToken: string | null;
      annotations: Annotation[];
      onSyncAnnotations?: () => Promise<void>;  // existing outbound sync function
      serverOrigin: string;  // base URL for API calls
    })
    ```

    Returns:
    - `state: ReviewState` -- tracks progress through sync -> submit flow
    - `pendingCount: number` -- count of unsynced annotations (those without github source and not yet mapped)
    - `error: string | null` -- error message if submission fails
    - `submitReview(event: ReviewEvent, body?: string): Promise<void>` -- the main action

    `submitReview` flow per D-14:
    1. Set state to 'syncing'
    2. If `pendingCount > 0` and `onSyncAnnotations` exists, call it to sync unsynced annotations
    3. Set state to 'submitting'
    4. POST to `{serverOrigin}/api/github/review` with body: `{ event, body, owner, repo, prNumber, token }`
    5. On success: set state to 'success'
    6. On error: set state to 'error', set error message
       - 401: "Session expired. Please re-authenticate." (clear token per D-14 from Phase 6 patterns)
       - 403: "You don't have permission to submit reviews on this PR"
       - 422: PR state issue -- parse message for closed/merged per D-39
       - 429: "Rate limit hit. Please try again later."
       - Network error: "Review submission failed. Check your connection."
    7. Return (caller handles success/close)

    `pendingCount` computation: count annotations where `source !== 'github-pr'` and `!isSummary` (regular local annotations that haven't been synced yet). This is an approximation -- the exact sync status depends on KV mappings on the server side, but this gives a useful count for the UI badge.
  </action>
