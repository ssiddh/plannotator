# Phase 7 — UI Design Contract

> Visual and interaction contract for thread management and resolution. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom theme system) |
| Preset | not applicable |
| Component library | none (hand-rolled components with Tailwind CSS v4) |
| Icon library | Inline SVG (heroicons-style, consistent with ToolbarButtons.tsx) |
| Font | Inter (sans), JetBrains Mono (mono) -- via `--font-sans` / `--font-mono` CSS vars |

**Source:** `packages/ui/theme.css` defines Tailwind bridge; `packages/ui/themes/plannotator.css` defines default theme tokens in oklch. Continues Phase 5/6 patterns.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, icon-to-text gaps |
| sm | 8px | Compact element spacing, button padding, modal field gaps |
| md | 16px | Default element spacing, panel padding, modal section padding |
| lg | 24px | Thread indentation per nesting level (continues Phase 5), section padding |
| xl | 32px | Layout gaps between major modal sections |

Exceptions:
- Summary left border width: 4px (D-21, left accent border)
- Touch target minimum: 32px (toolbar buttons, consistent with existing `p-1.5` pattern)
- Thread jump button size: 28px (compact navigation arrows in panel header)
- Modal textarea min-height: 120px (review body and summary input)

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (normal) | 1.5 |
| Label | 10px (`text-[10px]`) | 600 (semibold, `font-mono`) | 1.4 |
| Heading | 12px (`text-xs`) | 600 (semibold, `uppercase tracking-wider`) | 1.3 |
| Timestamp | 12px (`text-xs`) | 400 (normal) | 1.4 |

**Source:** Continues Phase 5 typography contract exactly. No new text roles introduced.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` | Page background, modal overlay, panel surfaces |
| Secondary (30%) | `var(--card)` / `var(--muted)` | Annotation cards, modal content area, thread containers |
| Accent (10%) | `var(--accent)` | Summarize button text, active thread jump button, thread picker selection highlight |
| Destructive | `var(--destructive)` | "Request Changes" button background |

Accent reserved for:
- "Summarize" button icon and text on hover (`text-accent`)
- Active thread in thread picker modal (`bg-accent/15 border-accent/30`)
- Thread jump button active state (`text-accent`)
- Export summaries button text (`text-accent`)

Additional semantic colors used:
- `var(--success)` -- "Approve" button background, "Resolved" badge background, resolution success toast
- `var(--warning)` -- Summary annotation background tint (`bg-warning/10`), summary left border (`border-warning`), "Summary" badge background (`bg-warning text-warning-foreground`)
- `var(--muted-foreground)` -- Resolved thread text at 70% opacity (D-17), disabled states, thread metadata text
- `var(--primary)` -- "Comment" button background (neutral review action)

## Interaction States

### Summarize Flow

```
[thread-parent-hover] --click Summarize--> [summary-modal]
                                              |
                                         type text
                                              |
                                              v
                                    [create-summary] --save--> [annotation-panel]
                                                                   |
                                                             user syncs
                                                                   |
                                                                   v
                                                         [synced-to-github]
                                                                   |
                                                          resolve mutation
                                                                   |
                                                                   v
                                                      [thread-resolved] (badge appears)
```

### Review Submission Flow

```
[review-tab-open] --fill optional body--> [ready]
                                            |
                                      click Approve/
                                      Request Changes/
                                      Comment
                                            |
                                            v
                                   [syncing-annotations]
                                            |
                                       sync complete
                                            |
                                            v
                                  [submitting-review]
                                            |
                                       success/fail
                                            |
                                            v
                                    [toast] --close modal--> [done]
```

| State | Review Tab Appearance |
|-------|----------------------|
| no-pr | Tab disabled, tooltip: "Link a PR to submit reviews" |
| ready | Buttons enabled, pending count shown |
| syncing | Buttons disabled, progress: "Syncing {N} annotations..." |
| submitting | Buttons disabled, progress: "Submitting review..." |
| success | Modal closes, success toast |
| error | Buttons re-enabled, error toast with retry |

### Thread Jump Navigation

| Interaction | Behavior |
|------------|----------|
| Click next | Find next thread root after current scroll position, scroll to it, highlight 1.5s |
| Click prev | Find prev thread root before current scroll position, scroll to it, highlight 1.5s |
| No more threads | Button disabled (opacity 50%) |
| Click resolved thread | Same behavior, regardless of resolved status |

### Resolution Status Sync

| Event | UI Update |
|-------|-----------|
| Inbound sync detects resolved thread | "Resolved" badge appears on thread parent (D-25) |
| Inbound sync detects re-opened thread | "Resolved" badge removed (D-30) |
| Outbound sync posts summary | Summary appears as reply on GitHub, thread resolves (D-09) |
| Resolution mutation fails | Toast: "Summary posted; thread not resolved -- resolve manually on GitHub" (D-11) |
| Permission denied | Toast per D-40, summary still syncs as reply (D-38) |
