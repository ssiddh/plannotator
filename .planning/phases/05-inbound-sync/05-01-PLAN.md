# Plan 01: Type Extensions + Server-Side Inbound Sync Endpoint

<objective>
Create the server-side inbound sync endpoint that fetches all PR comments from GitHub with pagination, deduplicates via KV mapping, detects edits and deletions, and returns categorized results. Extend shared types to support threading, edit detection, and the new response shape.

Purpose: Foundation for all sync operations -- client hooks and UI depend on this endpoint's response shape.
Output: Working `/api/pr/{pasteId}/sync/inbound` endpoint with pagination, deduplication, edit/delete detection. Extended types for threading and sync response.
</objective>

<action>
Create `packages/github/server/inboundSync.ts` with the core sync orchestration logic.

**Exports:**

```typescript
export async function performInboundSync(
  pasteId: string,
  prMetadata: PRMetadata,
  token: string,
  kv: any,
  options?: { since?: string }
): Promise<InboundSyncResponse>
```

**Implementation steps:**

1. **Fetch all comments** via `fetchPRComments(prMetadata, token, { since: options?.since })`.

2. **Process each comment** against KV mapping:
   - Call `getAnnotationId(pasteId, comment.id, kv)` for each comment.
   - If mapping exists AND comment is in the fetched results: check `updated_at` vs stored timestamp. If `updated_at` is newer, mark as "updated" and include in response with updated text (D-16).
   - If mapping exists AND comment is NOT in fetched results: mark as "deleted", add annotation ID to `deletedIds` (D-17). Call `deleteMapping(pasteId, annotationId, comment.id, kv)`.
   - If no mapping exists: this is a new comment. Generate a stable annotation ID using the pattern `gh-${comment.id}` (e.g., `gh-review_12345`). Call `setMapping(pasteId, annotationId, comment.id, kv, 30 * 24 * 60 * 60)` (30-day TTL). Mark as "new".
   - If mapping exists and `updated_at` is NOT newer: mark as "skipped".

3. **Deletion detection** (D-17): To detect deletions, we need to know ALL previously-imported comment IDs. Use a KV list key `sync:{pasteId}:imported` that stores a JSON array of all imported GitHub comment IDs. After each sync, update this list. On subsequent syncs, compare fetched comment IDs against this list to find deletions. Comments in the list but NOT in the fetched results are deleted.

4. **Convert to PRCommentForClient** for each new/updated comment:
   - `id`: the annotation ID (from KV mapping or newly generated)
   - `githubCommentId`: the original `comment.id` string
   - `blockId`: leave as empty string `""` -- client does line mapping (Research open question #3)
   - `type`: if `comment.comment_type === "issue"` then `"GLOBAL_COMMENT"`, else `"COMMENT"`
   - `text`: `comment.body`
   - `originalText`: if review comment with line: `"[Line ${comment.line}]"`, if issue comment: `"[General comment]"`, if review comment without line: `"[Line unmapped]"` (UI-SPEC copy)
   - `author`: `comment.author.username`
   - `avatarUrl`: `comment.author.avatar`
   - `githubCommentUrl`: `comment.github_url`
   - `createdAt`: `comment.created_at`
   - `updatedAt`: `comment.updated_at`
   - `inReplyToId`: `comment.in_reply_to_id || null`
   - `commentType`: `comment.comment_type`
   - `line`: `comment.line || null`

5. **Update sync state** via `setSyncState(pasteId, Date.now(), "inbound", kv, 30 * 24 * 60 * 60)`.

6. **Build and return** `InboundSyncResponse` with stats.

7. **Track newly created mappings within the sync operation** to prevent duplicates from concurrent processing (Pitfall 5 from RESEARCH.md). Use a local `Set<string>` of comment IDs processed in this operation.
</action>

2. **Add URL pattern** after the existing `PR_METADATA_PATTERN`:

```typescript
const PR_SYNC_INBOUND_PATTERN = /^\/api\/pr\/([A-Za-z0-9]{6,16})\/sync\/inbound$/;
```

3. **Add route handler** before the "Unknown route" comment at the end of the `handle` function. The route is `GET /api/pr/{pasteId}/sync/inbound`:

```typescript
const syncInboundMatch = url.pathname.match(PR_SYNC_INBOUND_PATTERN);
if (syncInboundMatch && req.method === "GET") {
  const syncPasteId = syncInboundMatch[1];

  // Extract and validate token
  const token = extractToken(req);
  if (!token) {
    return Response.json(
      { error: "Authentication required to sync PR comments" },
      { status: 401 }
    );
  }

  const authResult = await validateGitHubToken(token, kv);
  if (!authResult.valid) {
    return Response.json(
      { error: authResult.error || "Invalid or expired token" },
      { status: 401 }
    );
  }

  // Load PR metadata
  let syncPrMetadata: PRMetadata | null = null;
  if (kv) {
    const syncPrJson = await kv.get(`sync:${syncPasteId}:pr`);
    if (syncPrJson) {
      try { syncPrMetadata = JSON.parse(syncPrJson); } catch {}
    }
  }
  if (!syncPrMetadata && storage) {
    syncPrMetadata = await storage.getPRMetadata(syncPasteId);
  }
  if (!syncPrMetadata && kv) {
    const legacyJson = await kv.get(`pr:${syncPasteId}`);
    if (legacyJson) {
      try { syncPrMetadata = JSON.parse(legacyJson); } catch {}
    }
  }

  if (!syncPrMetadata) {
    return Response.json(
      { error: "No PR found for this paste" },
      { status: 404 }
    );
  }

  // Get last sync timestamp for incremental fetch
  const syncState = kv ? await getSyncState(syncPasteId, kv) : null;
  const since = syncState?.lastSyncTimestamp
    ? new Date(syncState.lastSyncTimestamp).toISOString()
    : undefined;

  try {
    const result = await performInboundSync(
      syncPasteId,
      syncPrMetadata,
      token,
      kv,
      { since }
    );
    return Response.json(result);
  } catch (e) {
    const msg = e instanceof Error ? e.message : "Sync failed";
    if (msg.includes("token_expired")) {
      return Response.json({ error: "token_expired" }, { status: 401 });
    }
    if (msg.includes("rate_limited")) {
      const resetTime = msg.split(":")[1];
      return Response.json(
        { error: "rate_limited", resetAt: resetTime },
        { status: 429 }
      );
    }
    return Response.json({ error: msg }, { status: 500 });
  }
}
```
</action>
