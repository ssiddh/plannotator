**1. `buildThreadTree(comments: PRCommentForClient[], blocks: Block[]): Annotation[]`**

Converts flat server-side `PRCommentForClient[]` into nested `Annotation[]` tree:

1. First pass: convert each `PRCommentForClient` to an `Annotation` object:
   - `id`: `comment.id` (the annotation ID from server)
   - `blockId`: Use `mapLineToBlock(comment.line, blocks)` if `comment.line !== null`. Fall back to `"global"` if `mapLineToBlock` returns null (D-18) or if `comment.line` is null.
   - `startOffset`: 0
   - `endOffset`: 0
   - `type`: `comment.type === "GLOBAL_COMMENT" ? AnnotationType.GLOBAL_COMMENT : AnnotationType.COMMENT`
   - `text`: `comment.text`
   - `originalText`: `comment.originalText`
   - `createdA`: `new Date(comment.createdAt).getTime()`
   - `author`: `comment.author`
   - `source`: `"github-pr"` (SYNC-IN-08)
   - `images`: `[{ path: comment.avatarUrl, name: \`github-avatar-${comment.author}\` }]`
   - `githubCommentUrl`: `comment.githubCommentUrl`
   - `startMeta`: `{ parentTagName: "span", parentIndex: 0, textOffset: 0 }`
   - `endMeta`: `{ parentTagName: "span", parentIndex: 0, textOffset: 0 }`
   Store in a Map keyed by `comment.githubCommentId`.

2. Second pass: group by `inReplyToId`:
   - If `comment.inReplyToId` is not null, find parent in Map by matching `inReplyToId` against `githubCommentId` values. If parent found, push this annotation to `parent.children` (create array if undefined). If parent not found (orphaned reply), treat as root.
   - If `comment.inReplyToId` is null, it's a root annotation.

3. Sort children within each parent by `createdA` ascending (SYNC-IN-06: chronological order).

4. Clamp depth to 3 levels max (UI-SPEC). In practice GitHub REST only gives 1 level of replies, but clamp defensively.

5. Return root annotations array.

```typescript
    month: "short",
    day: "numeric",
    hour: "numeric",
    minute: "2-digit",

}
```

Rewrite `packages/github/client/useGitHubPRSync.ts` to use the new server endpoint.

**Interface changes:**

```typescript
interface UseGitHubPRSyncOptions {
  pasteId: string;
  prMetadata: PRMetadataWithSync | null;
  blocks: Block[];
  token: string | null;
  pasteServiceUrl?: string;
  pollInterval?: number; // milliseconds, default 300000 (5 minutes per D-05)
  enabled?: boolean;
  onSyncComplete?: (stats: InboundSyncResponse["stats"]) => void;
  onError?: (error: string, type: "network" | "rate_limit" | "token_expired") => void;
}

interface UseGitHubPRSyncResult {
  annotations: Annotation[];
  isLoading: boolean;
  isSyncing: boolean; // true during active sync (vs initial load)
  error: string | null;
  lastSync: Date | null;
  newCommentCount: number; // Badge count for toolbar
  syncFromGitHub: () => Promise<void>; // Manual trigger
}
```

**Implementation:**

1. **State management:**
   - `annotations: Annotation[]` -- merged annotation state (thread tree built from flat data)
   - `isLoading` / `isSyncing` -- distinguish first load from subsequent syncs
   - `error: string | null`
   - `lastSync: Date | null`
   - `newCommentCount: number` -- count of comments imported in last sync
   - `knownCommentIds: Set<string>` via useRef -- local dedup guard (Pitfall 5)

2. **Core `fetchAndSync` function:**
   - Call `fetch(\`${pasteServiceUrl}/api/pr/${pasteId}/sync/inbound\`, { headers: { Authorization: \`Bearer ${token}\` } })`
   - Parse response as `InboundSyncResponse`
   - Call `buildThreadTree(response.annotations, blocks)` to get nested `Annotation[]`
   - Merge new annotations with existing state:
     - Add new annotations
     - Update text for updated annotations (match by `id`)
     - Remove annotations whose IDs are in `response.deletedIds` (D-17)
   - Update `newCommentCount` from `response.stats.new`
   - Update `lastSync` to `new Date()`
   - Call `onSyncComplete?.(response.stats)`

3. **Error handling with retry** (D-12):
   - Wrap `fetchAndSync` in retry logic: 3 attempts, delays of 1s, 2s, 4s (exponential backoff)

   **Concrete retry wrapper pattern:**
   ```typescript
   async function fetchWithRetry(): Promise<InboundSyncResponse> {
     let lastError: Error | null = null;
     for (let attempt = 0; attempt < 3; attempt++) {
       try {
         const response = await fetch(
           `${pasteServiceUrl}/api/pr/${pasteId}/sync/inbound`,
           { headers: { Authorization: `Bearer ${token}` } }
         );

         // Check response.status BEFORE parsing body
         if (response.status === 401) {
           // D-15: token expired -- do NOT retry
           throw Object.assign(new Error("token_expired"), { type: "token_expired" as const });
         }
         if (response.status === 429) {
           // D-13: rate limited -- do NOT retry automatically
           const body = await response.json();
           throw Object.assign(new Error(`rate_limited:${body.resetAt || ""}`), { type: "rate_limit" as const });
         }
         if (!response.ok) {
           throw new Error(`HTTP ${response.status}`);
         }

         return await response.json() as InboundSyncResponse;
       } catch (err: any) {
         // Non-retryable errors: propagate immediately
         if (err.type === "token_expired" || err.type === "rate_limit") {
           throw err;
         }
         lastError = err;
         if (attempt < 2) {
           await new Promise(r => setTimeout(r, 1000 * Math.pow(2, attempt))); // 1s, 2s, 4s
         }
       }
     }
     throw lastError || new Error("Sync failed after 3 retries");
   }
   ```

   - On 401 response: call `onError?.("token_expired", "token_expired")` -- do NOT retry (D-15)
   - On 429 response: parse `resetAt` from response body, call `onError?.(\`rate_limited:${resetAt}\`, "rate_limit")` -- do NOT retry automatically (D-13), but pause polling until reset time
   - On network error: retry with backoff. After 3 failures, call `onError?.("Sync failed. Check your connection.", "network")`

4. **Page Visibility API polling** (D-05, D-06):
   - Default `pollInterval` = 300000 (5 minutes)
   - On mount (if tab visible): do initial sync, then start interval
   - On `visibilitychange`:
     - `document.hidden === true`: clear interval (pause polling)
     - `document.hidden === false`: immediate sync + restart interval
   - Cleanup on unmount: clear interval, remove visibility listener

5. **Rate limit pause** (D-13):
   - When rate limit error received, store `rateLimitResetTime` in ref
   - Polling check: before each poll, check if `Date.now() < rateLimitResetTime`. If so, skip this poll cycle.
   - Clear `rateLimitResetTime` after reset time passes

6. **`syncFromGitHub` export:** Public function that calls `fetchAndSync()` directly (bypasses polling interval). Sets `isSyncing = true` during execution.
</action>
