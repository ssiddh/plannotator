```typescript
function formatAbsoluteTimestamp(isoStringOrMs: string | number): string {
  const date = typeof isoStringOrMs === "number"
    ? new Date(isoStringOrMs)
    : new Date(isoStringOrMs);
  return new Intl.DateTimeFormat(undefined, {
    month: "short",
    day: "numeric",
    hour: "numeric",
    minute: "2-digit",
  }).format(date);
}
```

e. **Add `selectedId` and `onSelect`/`onDelete`/`onEdit` as context or props for recursive rendering**: Since `AnnotationCard` is nested, the recursive children need access to `selectedId`, `onSelect`, `onDelete`. The cleanest way: change `AnnotationCard` to accept these as the existing pattern already provides them.

Actually, looking at the existing code, `AnnotationCard` already receives `onSelect`, `onDelete`, `onEdit` as props. For children, they need to use the parent's `onSelect`/`onDelete` callbacks with the child's ID. The rendering in the parent AnnotationPanel already handles this by calling `onSelect(ann.id)`. For children, we need to pass the panel-level callbacks. 

The simplest approach: the `onSelect` inside `AnnotationCard` is already a closure `() => onSelect(ann.id)` passed from the parent. For recursive rendering inside `AnnotationCard`, we need the original `onSelect(id)`, `onDelete(id)`, `onEdit(id, ...)` functions from the AnnotationPanel.

**Solution:** Add panel-level callbacks to AnnotationCard via additional props:

```typescript
const AnnotationCard: React.FC<{
  annotation: Annotation;
  isSelected: boolean;
  onSelect: () => void;
  onDelete: () => void;
  onEdit?: (updates: Partial<Annotation>) => void;
  depth?: number;
  selectedId?: string | null; // For recursive children
  onSelectById?: (id: string) => void; // For recursive children
  onDeleteById?: (id: string) => void; // For recursive children
  onEditById?: (id: string, updates: Partial<Annotation>) => void; // For recursive children
}>
```

In the AnnotationPanel rendering loop, pass these additional props:
```tsx
<AnnotationCard

```
