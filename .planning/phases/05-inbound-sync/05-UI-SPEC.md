# Phase 5 — UI Design Contract

> Visual and interaction contract for the inbound sync phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom theme system) |
| Preset | not applicable |
| Component library | none (hand-rolled components with Tailwind CSS v4) |
| Icon library | Inline SVG (heroicons-style, consistent with existing ToolbarButtons.tsx) |
| Font | Inter (sans), JetBrains Mono (mono) — via `--font-sans` / `--font-mono` CSS vars |

**Source:** `packages/ui/theme.css` defines Tailwind bridge; `packages/ui/themes/plannotator.css` defines default theme tokens in oklch.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Compact element spacing, button padding |
| md | 16px | Default element spacing, panel padding |
| lg | 24px | Thread indentation per nesting level (D-04), section padding |
| xl | 32px | Layout gaps between major sections |

Exceptions:
- Avatar size: 24px (D-07, inline with comment text)
- Badge diameter: 16px (notification count badge on sync button)
- Touch target minimum: 32px (toolbar buttons, consistent with existing `p-1.5` pattern)

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (normal) | 1.5 |
| Label | 10px (`text-[10px]`) | 600 (semibold, `font-mono`) | 1.4 |
| Heading | 12px (`text-xs`) | 600 (semibold, `uppercase tracking-wider`) | 1.3 |
| Timestamp | 12px (`text-xs`) | 400 (normal) | 1.4 |

**Source:** Matches existing AnnotationPanel patterns — `text-xs font-semibold uppercase tracking-wider` for section headings, `text-[10px] font-mono` for count badges. Label weight unified to semibold (600) for visual consistency with headings.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` | Page background, panel surfaces |
| Secondary (30%) | `var(--card)` / `var(--muted)` | Annotation cards, thread containers, badge backgrounds |
| Accent (10%) | `var(--accent)` | Sync button border/text when active, new comment badge background |
| Destructive | `var(--destructive)` | Error toasts only |

Accent reserved for:
- Sync button active state border and text (`bg-accent/15 text-accent border-accent/30` — matches FeedbackButton pattern)
- New comment count badge background (`bg-accent text-accent-foreground`)
- Clickable GitHub username links on hover

Additional semantic colors used:
- `var(--success)` — Sync complete toast icon
- `var(--warning)` — Partial sync warning toast, rate limit toast
- `var(--muted-foreground)` — Disabled sync button text, timestamp text, "syncing..." label
- `var(--primary)` — GitHub avatar ring/border on hover

## Interaction States

### Sync Button State Machine

```
[no-pr]  ───PR created───>  [idle]
                              │
                         click/poll
                              │
                              v
                          [syncing] ──success──> [idle] (badge updates)
                              │
                           failure
                              │
                              v
                         [error] ──retry──> [syncing]
                              │
                         3x fail
                              │
                              v
                      [error-toast] ──manual retry──> [syncing]
```

| State | Button Appearance | Badge |
|-------|-------------------|-------|
| no-pr | Disabled, muted, tooltip | Hidden |
| idle | Active, accent styling | Shows count if > 0 |
| syncing | Icon spinning, "Syncing..." label | Previous count |
| error | Active (retry available) | Previous count |
| error-toast | Active (retry available) | Previous count |

### Auto-Poll Behavior

| Condition | Behavior |
|-----------|----------|
| PR exists + tab visible | Poll every 5 minutes (D-05) |
| PR exists + tab hidden | Polling paused (D-06) |
| Tab becomes visible | Immediate sync + resume polling |
| No PR | No polling |
| Rate limited | Pause polling until `X-RateLimit-Reset` (D-13) |

### Annotation Panel Threading

| Interaction | Behavior |
|------------|----------|
| Click annotation card | Scroll plan viewer to annotation position |
| Click GitHub username | Open GitHub comment in new tab (D-10) |
| Hover avatar | Scale to 1.1x (existing `.viewer-avatar:hover` pattern) |
| Local annotation on same line as GitHub annotation | Both displayed, local is editable, GitHub is read-only (D-08) |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Sync from GitHub" (toolbar button desktop label) |
| Primary CTA mobile | SVG sync icon only (no text) |
| Empty state heading | "No GitHub comments yet" |
| Empty state body | "Comments posted on the linked PR will appear here after syncing." |
| Disabled tooltip | "Create a PR first to sync comments" (D-03, exact text) |
| Sync success toast | "Synced {N} comments from GitHub" |
| Sync success (no new) | "Already up to date" |
| Sync partial warning | "Imported {N} of {M} comments (page {P} failed -- retrying)" (D-14) |
| Rate limit toast | "Rate limit hit. Retrying at {time}" (D-13) |
| Network error toast | "Sync failed. Check your connection." with "Retry" button |
| Token expired toast | "Session expired. Please sign in again." (auto-redirects per D-15) |
| Syncing indicator | "Syncing..." (replaces button label during fetch) |
| Retrying indicator | "Retrying... ({N}/3)" (D-12) |
| Badge format | "{N}" for 1-9, "9+" for 10+ |
| Global annotation label | "[General comment]" (for issue comments without line position) |
| Unmapped annotation label | "[Line unmapped]" (for drifted line comments placed globally per D-18) |
