    export async function performOutboundSync(
      pasteId: string,
      annotations: ExportAnnotation[],
      blocks: ExportBlock[],
      planMarkdown: string,
      prMetadata: PRMetadataWithSync,
      token: string,
      kv: any,
      options?: {
        fetchFn?: typeof fetchPRComments;

    **Implementation steps (per D-01 through D-21):**

    1. **Drift detection (D-05, D-06):** Call `generatePlanHash(planMarkdown)` and compare to `prMetadata.planHash`. Set `hasDrift = hashes differ`. Do NOT block sync — just report.

    2. **Classification (D-01, D-02, D-04, D-07, D-19):**
       - Filter out GLOBAL_COMMENT annotations. Count them. Add warning if count > 0: `"{N} global annotations skipped (no line position)"`.
       - For remaining annotations, count those with `images?.length > 0`. Add warning if count > 0: `"{N} annotations with images synced text only"`.
       - Fetch current GitHub comment state via `fetchPRComments(prMetadata, token)`.
       - Build a Map from comment ID to comment body: `commentBodyMap`.
       - For each non-global annotation:
         - Call `getCommentId(pasteId, annotation.id, kv)`.
         - If null: classify as NEW.
         - If exists: look up `commentBodyMap.get(commentId)`. If body differs from `annotation.text` (for COMMENT type) or differs from the expected suggestion format (for DELETION type): classify as EDITED with `githubCommentId`.
         - If exists and body matches: classify as SKIPPED.
       - NOTE on edit comparison for DELETION: DELETION annotations produce suggestion block format (`> originalText\n\n\`\`\`suggestion...`). Compare GitHub body to the expected suggestion format, not raw annotation.text.

    3. **Post new annotations (D-17, D-11, D-18):**
       - Call `mapAnnotationsToComments(newAnnotations, blocks, "plans/${pasteId}.md")` to get `ReviewComment[]`.
       - If comments.length > 0: call `submitBatchReview(owner, repo, prNumber, token, comments, "Annotations synced from Plannotator")` with event "COMMENT".
       - Review body text: `"Annotations synced from Plannotator"`.

    4. **Recover comment IDs (Pitfall 1 from RESEARCH.md):**
       - After submitBatchReview returns `reviewResponse`, extract `reviewResponse.id` (the review ID).
       - Call `githubRequest("GET /repos/${owner}/${repo}/pulls/${prNumber}/reviews/${reviewId}/comments", token)` to get array of comment objects with numeric `.id`, `.body`, `.path`, `.line`.
       - Match returned comments to submitted annotations by `(path, line, body)` tuple. If ambiguous (multiple annotations on same line), fall back to positional order (comments in submission order).
       - For each matched pair: call `setMapping(pasteId, annotationId, String(commentObject.id), kv, KV_TTL)` where `KV_TTL = 30 * 24 * 60 * 60`.

    5. **Post edit replies (D-03):**
       - For each edited annotation:
         - Extract numeric ID by stripping "review_" prefix from `githubCommentId`: `const numericId = githubCommentId.replace(/^review_/, "")`.
         - Call `githubRequest("POST /repos/${owner}/${repo}/pulls/${prNumber}/comments/${numericId}/replies", token, { body: "Updated: ${annotation.text}" })`.
       - Count successful replies as `editCount`.

    6. **Update sync state (D-15):**
       - Call `setSyncState(pasteId, Date.now(), "outbound", kv, KV_TTL)`.

    7. **Return result:**
       ```typescript
       return { syncedCount: newAnnotations.length, editCount, skippedCount, warnings, hasDrift };
       ```

    Parse `owner` and `repo` from `prMetadata.repo.split("/")`. Use `prMetadata.pr_number` for PR number.

    Use dependency injection via `options` parameter for all external calls (fetchPRComments, githubRequest, submitBatchReview, generatePlanHash) to enable testing. Default to real implementations when options not provided.
