# Phase 6 — UI Design Contract

> Visual and interaction contract for the outbound sync phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom theme system) |
| Preset | not applicable |
| Component library | none (hand-rolled components with Tailwind CSS v4) |
| Icon library | Inline SVG (heroicons-style, consistent with existing ToolbarButtons.tsx) |
| Font | Inter (sans), JetBrains Mono (mono) — via `--font-sans` / `--font-mono` CSS vars |

**Source:** `packages/ui/theme.css` defines Tailwind bridge; `packages/ui/themes/plannotator.css` defines default theme tokens in oklch. Inherited from Phase 5 UI-SPEC — no changes.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Compact element spacing, button padding |
| md | 16px | Default element spacing, drift banner padding |
| lg | 24px | Section padding, banner content gap |
| xl | 32px | Layout gaps between major sections |

Exceptions:
- Badge diameter: 16px (notification count badge on outbound sync button, matches Phase 5 inbound badge)
- Touch target minimum: 32px (toolbar buttons, consistent with existing `p-1.5` pattern)
- Drift banner action button padding: 12px horizontal (slightly wider than standard sm for hit target)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|

| Timestamp | 12px (`text-xs`) | 400 (normal) | 1.4 |

**Source:** Matches Phase 5 UI-SPEC exactly. No new typography roles introduced.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` | Page background, panel surfaces |
| Secondary (30%) | `var(--card)` / `var(--muted)` | Toast backgrounds, drift banner background |
| Accent (10%) | `var(--accent)` | Outbound sync button border/text when active, unsynced count badge background |
| Destructive | `var(--destructive)` | Error toasts only |

Accent reserved for:
- Outbound sync button active state border and text (`bg-accent/15 text-accent border-accent/30` — matches SyncButton and FeedbackButton pattern)
- Unsynced annotation count badge background (`bg-accent text-accent-foreground`)

Additional semantic colors used:
- `var(--success)` — Sync complete toast icon
- `var(--warning)` — Drift warning banner border and icon, partial sync warning toast, image-stripped warning toast
- `var(--muted-foreground)` — Disabled sync button text, "Syncing..." label, drift banner secondary text
- `var(--primary)` — Not used by new Phase 6 components

## Interaction States

### Outbound Sync Button State Machine

```
[no-pr]  ───PR created───>  [idle]
                              │
                         click sync
                              │
                              v
                      [drift-check] ──no drift──> [syncing]
                              │
                        drift detected
                              │
                              v
                      [drift-warning] ──proceed──> [syncing]
                              │                      │
                           cancel               success/failure
                              │                      │
                              v                      v
                          [idle]              [idle] / [error]
                                                │
                                           3x fail
                                                │
                                                v
                                         [error-toast] ──manual retry──> [syncing]
```

| State | Button Appearance | Badge |
|-------|-------------------|-------|
| no-pr | Disabled, muted, tooltip: "Create a PR first to sync annotations" | Hidden |
| idle | Active, accent styling | Shows unsynced count if > 0 |
| drift-check | Active (brief, checking hash) | Previous count |
| drift-warning | Active (user deciding) | Previous count |
| syncing | Icon spinning, "Syncing..." label | Previous count |
| error | Active (retry available) | Previous count |
| error-toast | Active (retry available) | Previous count |

### Error Handling Interactions

| Error Type | Behavior |
|------------|----------|
| Network error (1st-3rd attempt) | Button shows spinning icon, toast shows "Retrying... (N/3)" (D-12) |
| Network error (after 3 failures) | Error toast with "Retry" button, button returns to idle (D-12) |
| Rate limit (429) | Toast: "Rate limit hit. Retrying at {time}", auto-retry after reset (D-13) |
| Auth failure (401/403) | Clear token, redirect to OAuth with return_to URL (D-14) |
